## Color Theory üé®

You‚Äôve probably seen a variation of the image below at least once in your life. If not, that‚Äôs completely fine!

![image](https://firebasestorage.googleapis.com/v0/b/codedex-io.appspot.com/o/blogs%2Fyour-website-glow-up-guide%2Fcolor-wheel.jpg?alt=media&token=a130fc3e-480f-407c-89cc-29ac8fc5bebb)

This is a **color wheel**. It‚Äôs your cheat sheet for choosing colors that actually look good together. 

Color wheels are broken down into primary (red, blue, yellow), secondary (green, orange, purple, which are a mix of primary colors), and tertiary colors (a mix of primary and secondary colors).

Certain parts of the wheel naturally pair well together, and those pairings are what help you build a strong color palette.

Here are a few beginner-friendly color combinations: 

- **Complementary**: Opposite on the wheel, high contrast (ex. red + green üéÑ)
- **Analogous**: Side-by-side, harmonious (ex. yellow + yellow-orange + orange üåû)
- **Triadic**: Evenly spaced and form a triangle (ex. red + blue + yellow )
- **Monochromatic**: Different tints/shades of the same base color  (great for minimal sites)

### Okay, so why is the color wheel important?

The colors you choose set the mood of your entire website. With a basic understanding of the color wheel, you can build an aesthetically pleasing palette that fits your website‚Äôs vibe. Color theory helps you avoid random combinations that clash and instead gives your site a balanced and intentional look.

### A quick note on black and white

Black and white are neutral colors, meaning they pretty much work with everything!

### Gradients

A [gradient](https://en.wikipedia.org/wiki/Color_gradient) is a visual map of colors that blend between each other gradually. [CSS Gradients](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Images/Using_gradients) are a simple addition to your website to give you more depth! Gradients can be used for any components on your website, such as background, buttons, text, and more.

There are three main types of gradients: **linear**, **radial**, and **conic**.

![image](https://firebasestorage.googleapis.com/v0/b/codedex-io.appspot.com/o/blogs%2Fyour-website-glow-up-guide%2Fgradient-types.png?alt=media&token=f6d5d61b-7498-4030-9ce3-b9a15cb4787d)

They are super easy to implement! 

For example, to achieve the gradients shown in the pictures above:

```css
background: linear-gradient(180deg, #FF87AB 0%, #FFF585 100%);

background: radial-gradient(50% 50% at 50% 50%, #0F9 0%, #A21DDB 100%);

background: conic-gradient(from 180deg at 50% 50%, #FF2B2B 0deg, #FFA100 67.5deg, #FFF585 138.4615409374237deg, #0F9 209.4230818748474deg, #3613FE 283.8461637496948deg, #A21DDB 358.2692241668701deg);
```

## Typography

[Typography](https://en.wikipedia.org/wiki/Typography) is the art of styling text. Fonts are one of the easiest ways to change a website‚Äôs entire vibe. ‚òÆÔ∏è

Bold, soft, playful, minimal, the font you choose instantly sets the tone.

### Why typography matters üó£Ô∏è

Typography decides your website‚Äôs vibe.

It tells people whether your site feels:
- playful 
- minimal 
- bold 
- techy 
- elegant
- chaotic (intentionally or not üò≠)

Even if your colors look amazing, bad typography can make a site feel messy or unfinished. 

Keep it simple! Try to keep the number of fonts used on your website to a minimum, with different font weights for variation. A [sans-serif](https://en.wikipedia.org/wiki/Sans-serif) + [serif](https://en.wikipedia.org/wiki/Serif) combo usually looks super clean. üßº

Need typography pairing help? 
- [Font Pair](https://www.fontpair.co/)
- [Font Joy](https://fontjoy.com/)
- [Font Mixer](https://fontmixer.app/)

Once you find the desired fonts for your website, you can get an embed code from [Google Fonts](https://fonts.google.com/) and import them into your HTML by pasting this code into the head of the document.

```html
<link href="https://fonts.googleapis.com/css2?family=Poppins:wght@300;600&display=swap" rel="stylesheet">
```

To apply the font, you can declare the font using the `font-family` in the CSS:

```css
body {
  font-family: 'Poppins', sans-serif;
}
```

## Hover Effects

Okay, now your website colors look pretty. ‚ú®

Did you know that you can make components on your website change when you hover over them? üòÆ If you didn‚Äôt, let‚Äôs get into it!

To create hover effects in CSS, we use pseudo-classes. A **pseudo-class** lets you style a specific state of a selected element. Pseudoclasses consist of a colon (`:`) followed by a class name, in this case, `hover`.

First, style your element normally. Then, when you want it to react, you can add a `hover` class to change its color, add a drop shadow, glow, bounce, and more! 

### Hover example

```css
.kitty-wrapper:hover .spin {
  opacity: 1;
}

.kitty-wrapper:hover .default {
  opacity: 0;
}
```

![image](https://firebasestorage.googleapis.com/v0/b/codedex-io.appspot.com/o/blogs%2Fyour-website-glow-up-guide%2Fhover.gif?alt=media&token=aeb38709-3c34-4c34-aa96-ab9ac9100b0a)

In the code above, I added the [`:hover`](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Selectors/:hover) to my original class, so whenever I hover over the cat, the spinning cat fades in by increasing its opacity while the default cat fades out. When you stop hovering, everything goes back to normal. 

Pretty cool, right? üòÜ

## Keyframes

Ready to make your website move? üìª

CSS **keyframes** are like choreography for your website. üíÉüï∫

Unlike hover, keyframes can run automatically, loop forever, or run when triggered using JavaScript! With keyframes, you can decide exactly how elements change at different points in time. ‚è∞

### How do keyframes work?

```css
@keyframes animation-name {
  0% { /* initial state */ }
  50% { /* middle state */ }
  100% { /* final state */ }
}
```

Each percentage tells what you want to happen to your element. 

```css
selector {
  animation: animation-name duration easing-type iteration-count;
}
```

Next, you can add the animation to your CSS ID or class! 

Specify the animation name you defined with `@keyframes` length of the animation, the easing type, the animation transition type and if you want your animation to run a finite or infinite number of times. 

Voila! You just created an animation in CSS! üßë‚Äçüç≥

### Keyframe example

What if we wanted to make Moodeng, a chaotic little hippo, bounce? ü¶õ‚ú®

We have to create a keyframes selector that moves her 20 pixels down halfway through the animation and returns Moodeng to the original spot at 100% completion, so it looks like she jumps!

```css
@keyframes moodeng-bounce {
  0% { transform: translateY(0); }
  50% { transform: translateY(-20px); }
  100% { transform: translateY(0); }
} 
```

Next, we have to 

```css
.moodeng {
  width: 300px;
  animation: moodeng-bounce 0.8s infinite ease-in-out;
}
```

Add the animation to the `moodeng` selector. I set it to 0.8 seconds and gave it infinite bouncing time with an ease-in and out effect.

## Scrolling Effects

Ever click a link in your navbar and have the page jump instantly to a section? üò¨ It works, but it can feel a little jarring. 

Smooth scrolling works with anchor links that are defined in the HTML


```html
<a href="#section-id">`
```

These links take you to different areas of your page in a smooth, polished way.

All you need is a tiny bit of CSS:

```css
html{
  scroll-behavior: smooth;
}
```

Now, when a link is clicked, the page scrolls into view naturally, so your visitors can see exactly _where_ the page has taken them.

![image](https://firebasestorage.googleapis.com/v0/b/codedex-io.appspot.com/o/blogs%2Fyour-website-glow-up-guide%2Fscrolling.gif?alt=media&token=dd0a2015-422b-49d2-88a2-c6a50e6c16a8)

See how my page smoothly scrolled to the section when a navigation link was pressed? All of this is done in just **3 lines of CSS**. üòâ
